# Toast

Temporary notification that appears at the bottom of the screen and auto-dismisses after a timeout.

## When to use

| Use Toast when... | Don't use Toast when... |
|---|---|
| Confirming a completed action ("Bet placed successfully") | The message requires user action — use Alert or Dialog |
| Showing non-critical, transient feedback | The information must persist until acknowledged |

<Canvas of={ToastStories.AllVariants} />

## Best practices

### Do
- Keep messages to a single line — concise and scannable
- Use the appropriate status variant (success, error, info, warning)

### Don't
- Stack more than 3 toasts simultaneously
- Use toasts for errors that require user intervention

## Accessibility
- Toasts use `role="status"` with `aria-live="polite"` for screen reader announcements
- Dismiss timeout is paused on hover/focus to give users time to read
