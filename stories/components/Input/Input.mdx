# Input

A text field for collecting user input. Supports multiple types (text, search, password, currency, phone, checkbox) through the `type` prop.

## When to use

| Use Input when... | Don't use Input when... |
|---|---|
| Collecting text, email, or numeric data | Selecting from predefined options (use Dropdown) |
| Search functionality | Toggling a binary state (use Toggle/Checkbox) |
| Password entry | Long-form text (use InputTextArea) |
| Currency amounts | Date selection (use DatePicker) |

## Composition with InputField

<Canvas of={InputStories.WithLabel} />

For labeled inputs with descriptions, compose with:

```tsx
<InputField id="field">
  <InputFieldTitle id="title">Label</InputFieldTitle>
  <Input id="input" name="field" />
  <InputFieldDescription id="desc">Helper text</InputFieldDescription>
</InputField>
```

## Best practices

### Do

- Always pair with `InputFieldTitle` for accessibility
- Use `placeholder` to show expected format, not as a label
- Use `type="search"` for search fields (adds clear button)
- Add `InputFieldDescription` for complex validation rules

### Don't

- Don't use Input for multi-line text (use InputTextArea)
- Don't hide the label — placeholder disappears on focus
- Don't use `type="number"` for phone numbers (use `type="tel"`)
- Don't forget `name` prop — it's required for form submission

## Accessibility

- Label connected via `InputFieldTitle` → `htmlFor`
- Description linked via `aria-describedby`
- Search type includes clear button with keyboard support
- Password type includes accessible show/hide toggle
- Disabled state communicated to assistive technology
